# Чек-лист: зібрати свого HR-бота

> Покроково, з типовими помилками і fallback біля кожного кроку.
> Помилки виписані рівно ті, що ламають бота найчастіше. Йди по порядку - не перестрибуй.

---

## Етап 0 - Підготовка (10 хв)

- [ ] **Акаунт n8n** (Cloud або self-hosted). Якщо self-hosted - запущений і відкривається.
- [ ] **Акаунт Supabase** (безкоштовний tier) - проєкт створено.
- [ ] **OpenAI API key** - є ключ з балансом (платний акаунт, не free trial - free часто без доступу до embeddings).
- [ ] **1-3 реальні HR-документи** твоєї компанії у форматі .md / .txt / .pdf (можна знеособлені).

> ⚠️ **Помилка:** OpenAI free trial без платіжки → embeddings падають з 429/403.
> ✅ **Fix:** прив'яжи картку до OpenAI, поповни на $5 - вистачить надовго.

---

## Етап 1 - Налаштувати Supabase (5 хв)

- [ ] Supabase → SQL Editor → New query
- [ ] Вставити ВЕСЬ `supabase-setup.sql` → Run
- [ ] Перевірити: `select * from documents limit 5;` → таблиця існує (порожня - ок)
- [ ] Скопіювати з Settings → API: `Project URL` + `service_role` key (НЕ anon!)

> ⚠️ **Помилка 1:** забув `create extension vector` → нода Supabase падає «type vector does not exist».
> ✅ **Fix:** SQL-скрипт уже містить `create extension` першим рядком. Просто виконай його повністю.

> ⚠️ **Помилка 2:** скопіював `anon` key замість `service_role` → запис не проходить (немає прав).
> ✅ **Fix:** бери саме `service_role` (Settings → API → Project API keys → service_role).

---

## Етап 2 - Імпортувати workflow у n8n (5 хв)

- [ ] n8n → Workflows → Import from File (або вже залиті: «HR-бот · Workflow 1» і «Workflow 2»)
- [ ] Імпортувати обидва: `workflow-1-ingestion.json` і `workflow-2-bot.json`

> ⚠️ **Помилка:** імпортував тільки один workflow і дивуєшся, що бот «нічого не знає».
> ✅ **Fix:** їх ДВА. Workflow 1 наповнює базу, Workflow 2 - сам бот. Спочатку 1, потім 2.

---

## Етап 3 - Підключити credentials (5 хв)

- [ ] **OpenAI credential:** у нодах Embeddings і Chat Model → вставити OpenAI API key
- [ ] **Supabase credential:** у нодах Supabase → вставити Project URL + service_role key
- [ ] Перевірити, що credential підхопився в УСІХ нодах (їх кілька: embeddings у двох місцях, Supabase у двох)

> ⚠️ **Помилка:** підключив credential в одній ноді, забув в іншій → частина workflow червона.
> ✅ **Fix:** пройди по всіх нодах embeddings/Supabase, переконайся - credential скрізь обраний.

---

## Етап 4 - Наповнити базу (Workflow 1) (5 хв)

- [ ] Відкрити Workflow 1 (наповнення)
- [ ] У ноді завантаження вказати свій HR-документ (файл / drag-drop / джерело)
- [ ] Натиснути **Execute Workflow**
- [ ] Дочекатись, поки всі ноди позеленіють
- [ ] Перевірити у Supabase → Table Editor → `documents` → з'явились рядки з текстом і вектором

> ⚠️ **Помилка 1:** розмірність вектора не збігається (помилка про dimension 1536 vs N).
> ✅ **Fix:** `text-embedding-3-small` дає рівно 1536; таблиця в SQL під 1536. Якщо міняв модель - онови `vector(N)` у SQL.

> ⚠️ **Помилка 2:** база лишилась порожня після Execute.
> ✅ **Fix:** перевір, що документ реально завантажився у нода-лоадер (не порожній вхід), і credential Supabase правильний.

- [ ] Повторити цей етап для КОЖНОГО документа (запускати по разу на файл)

---

## Етап 5 - Запустити бота (Workflow 2) (5 хв)

- [ ] Відкрити Workflow 2 (бот)
- [ ] Перевірити **embedding-модель** у ноді retrieve = ТА САМА, що у Workflow 1 (`text-embedding-3-small`)
- [ ] Відкрити Chat (кнопка Chat внизу / посилання Chat Trigger)
- [ ] Поставити тестові питання:
  - [ ] «Як оформити відпустку?» → відповідь з документа
  - [ ] «Скільки днів відпустки належить?» → конкретна цифра з твого документа
  - [ ] «Хто підписує заяву?» → ім'я/посада з документа
  - [ ] **Контрольне:** «Яка погода завтра?» → бот має відмовитись (не вигадати)

> ⚠️ **Помилка №1 усіх новачків:** бот відповідає «не знайшов» на правильні питання.
> ✅ **Fix:** РІЗНІ embedding-моделі при записі (WF1) і пошуку (WF2). Постав однакову - і пошук запрацює. Це причина #1 «бот тупий».

> ⚠️ **Помилка 2:** бот вигадує і відповідає на питання поза базою (погода).
> ✅ **Fix:** перевір системний промпт агента - має містити «відповідай ТІЛЬКИ з документів + не знаю → до HR».

---

## Етап 6 - Готово до здачі на хакатон

- [ ] Бот відповідає на 5 реальних питань з твоїх документів
- [ ] Відповіді правильні і вказують джерело (документ)
- [ ] На питання поза базою бот чесно каже «не знаю → до HR»
- [ ] Це твій реальний процес, не вчебний приклад

---

## Зведена таблиця типових помилок

| Симптом | Причина | Fix |
|---|---|---|
| «type vector does not exist» | не виконав create extension | запусти весь SQL-скрипт |
| запис у базу не проходить | взяв anon замість service_role | візьми service_role key |
| dimension mismatch (1536) | модель/таблиця не збігаються | text-embedding-3-small = 1536 |
| embeddings падають 429/403 | OpenAI free без платіжки | прив'яжи картку, поповни $5 |
| бот «нічого не знає» | імпортував 1 workflow з 2 | імпортуй обидва, запусти WF1 |
| **бот «не знайшов» на все** | **різні embedding-моделі WF1/WF2** | **постав однакову модель** |
| бот вигадує / відповідає на погоду | слабкий системний промпт | додай «тільки з документів» |

> Застряг попри fix - напиши, розберемо на наступній зустрічі.
