# Урок 4 - Роздатка: збери свого HR-бота

Усе, щоб зібрати робочого HR/onboarding-бота сьогодні ввечері. Імпортуєш workflow → підставляєш свої ключі → завантажуєш свої документи → бот відповідає.

---

## Що в пакеті

| Файл | Що це | Коли відкривати |
|---|---|---|
| **CHECKLIST-build-bot.md** | покроковий чек-лист зборки з типовими помилками і fix | головний файл, відкрий першим |
| **supabase-setup.sql** | SQL для Supabase: pgvector + таблиця + match_documents | Етап 1 чек-листа |
| **workflow-1-ingestion.json** | n8n workflow №1: наповнення бази знань | Етап 2 (імпорт) |
| **workflow-2-bot.json** | n8n workflow №2: сам бот (RAG) | Етап 2 (імпорт) |
| **sample-docs/** | 5 готових HR-документів Profigent для тренування | Етап 4 (якщо нема своїх) |

---

## Швидкий старт (за 6 кроків)

1. **Supabase:** створи проєкт → SQL Editor → встав `supabase-setup.sql` → Run.
2. **n8n:** імпортуй обидва workflow (`workflow-1-ingestion.json`, `workflow-2-bot.json`).
3. **Credentials:** встав свої OpenAI key + Supabase (URL + service_role) у відповідні ноди.
4. **Наповни базу:** у Workflow 1 вкажи свій HR-документ → Execute. Повтори на кожен документ.
5. **Запусти бота:** відкрий Workflow 2 → Chat → постав 5 питань.
6. **Перевір контрольне:** спитай щось поза базою (погода) → бот має відмовитись, не вигадати.

> Детально кожен крок + помилки - у **CHECKLIST-build-bot.md**.

---

## Стек (нагадування з лекції)

- **n8n** - оркестратор (з'єднує все, працює 24/7)
- **Supabase (pgvector)** - векторна база (persistent, дані під контролем)
- **OpenAI text-embedding-3-small** - текст → вектори (1536 розмірність)
- **GPT-4o-mini** - мозок (формулює відповідь з документів)
- **n8n Chat Trigger** - канал (вбудований веб-чат, без бот-токенів)

**Головне правило:** embedding-модель при ЗАПИСІ (WF1) і при ПОШУКУ (WF2) - однакова. Інакше бот «нічого не знаходить» (помилка #1 новачків).

5 готових HR-документів Profigent - використовуй для тренування, якщо своїх ще нема під рукою:

## Домашка-хакатон

1. Візьми **1 реальний HR-процес** твоєї компанії (онбординг / FAQ / політики / скринінг).
2. Збери **1-3 справжні документи** (можна знеособлені).
3. Зібери бота за чек-листом.
4. Постав 5 реальних питань, які працівники справді ставлять.
5. **На наступній зустрічі - демо + зворотний зв'язок.**

**Критерії оцінки:** працює (0-3) · точність з документів (0-3) · чесність «не знаю → до HR» (0-2) · вказує джерело (0-1) · реальний процес (0-1). Прохідний: працює + не галюцинує на контрольному питанні.

> Застряг - пиши, розберемо на зустрічі. Мета не ідеальний бот, а РОБОЧИЙ під твій справжній процес.
